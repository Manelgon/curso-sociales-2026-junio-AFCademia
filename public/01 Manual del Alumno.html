00Bienvenida

Hola, y gracias por estar aquí.

En los próximos 60 minutos vas a construir — desde una pantalla en blanco — un sistema que lee, clasifica y responde tus emails automáticamente. No vamos a importar nada. Vas a hacer cada nodo tú mismo.

¿Por qué construirlo a mano?

Porque cuando entiendes cómo está hecho, sabes modificarlo. Sabes ampliarlo. Sabes arreglarlo cuando algo falla. Importar un blueprint te da el resultado; construirlo te da el conocimiento.

Al terminar tendrás

  • Un escenario en Make funcionando, montado por ti, con 11 módulos conectados.
  • Tu bandeja organizada en 5 carpetas/etiquetas automáticamente.
  • Borradores de respuesta listos para revisar, redactados por la IA.
  • El criterio para escoger entre 4 variantes (Gmail/Outlook × OpenAI/Groq/Gemini).

01Módulo 1 · Conceptos

¿Qué es Make y cómo funciona?

Antes de tocar nada, tres ideas que te van a ahorrar mucha confusión.

Make.com en una frase

Make es como tener un becario que nunca duerme entre tus aplicaciones. Le dices: “cuando llegue un email, léelo, y según de qué trate, muévelo a una carpeta y déjame un borrador”. Y lo hace. Cada vez. Para siempre.

Tres palabras que vas a oír mucho

Escenario
El “flujo de trabajo” completo. Es el plano que tú vas a dibujar en Make.
Módulo
Cada paso del escenario. Por ejemplo: “leer email de Gmail”, “preguntar a la IA”, “mover a carpeta”.
Conexión
El permiso que le das a Make para entrar en tu Gmail, OpenAI, etc. Se hace una sola vez por servicio.

02Módulo 2 · Preparativos

Cuentas, carpetas y claves antes de empezar.

Esto es 10 minutos de trastear, pero solo se hace una vez en la vida.

Lista de la compra

1

Cuenta de Make.com

Entra en make.com y crea cuenta gratuita. El plan Free te da 1.000 operaciones al mes.

2

Tu cuenta de email del despacho

Gmail (Google Workspace) o Outlook (Microsoft 365). La que uses cada día.

3

Una clave de API de IA (elige UN proveedor)

La API key es la llave que identifica tu cuenta de IA. Es una cadena larga (50+ caracteres) que se ve algo así: sk-proj-aB3f9X2k7Lp_qMnT5vR8eW... Trátala como una contraseña: no la compartas y guárdala bien (gestor de contraseñas o el cuaderno). Solo se muestra UNA vez tras crearla.

3a · Cómo sacar la API key paso a paso

OPENAI platform.openai.com/api-keys

Para qué: usar modelos GPT (gpt-4.1-nano, gpt-4o-mini…). Calidad alta, coste por uso.

PasoAcción
1Abre platform.openai.com e inicia sesión (o crea cuenta).
2Arriba a la derecha → tu avatar → Your profile → menú lateral API keys.
3Botón “+ Create new secret key”.
4Nombre: Make-Curso. Permissions: All. Pulsa Create.
5Pulsa “Copy” y pégala YA en tu gestor de contraseñas o cuaderno.
€
OpenAI requiere saldo mínimo

Antes de que la key funcione tienes que añadir saldo (mín. ~5 €) en Billing → Payment methods → Add to credit balance. Si no, te dará error insufficient_quota.

GROQ console.groq.com/keys

Para qué: usar Llama 3.1 vía Groq. Muy rápido y gratis con cuota generosa. Ideal para empezar.

PasoAcción
1Abre console.groq.com → Sign in with Google (o email).
2Menú lateral izquierdo → API Keys.
3Botón “Create API Key”. Nombre: Make-Curso.
4Pulsa Submit → cópiala y guárdala. No se vuelve a ver.

→ El formato empieza por gsk_.

GEMINI aistudio.google.com/apikey

Para qué: usar Google Gemini. Gratis en el tier de Google AI Studio.

PasoAcción
1Abre aistudio.google.com/apikey con tu cuenta Google.
2Acepta los términos si te los pide.
3Botón “Create API key” → elige un proyecto (o “Create API key in new project”).
4Pulsa Copy y guárdala.

→ El formato empieza por AIza.

3b · Conecta tus cuentas dentro de Make

Cada servicio externo que use Make (Gmail, Outlook, OpenAI, Groq, Gemini) necesita una conexión. Se crea una sola vez y queda guardada para reutilizarse en todos tus escenarios. Hay dos tipos:

  • OAuth (Gmail, Outlook): Make abre una ventana del navegador, inicias sesión y das permisos. No tocas claves.
  • API key (OpenAI, Groq, Gemini): pegas en un campo la cadena que copiaste antes.
EMAIL Conexión Gmail · Outlook (OAuth)

Cuándo: la primera vez que añades un módulo de Gmail o Microsoft 365 Email (en el Trigger del Módulo 3, y de nuevo en “Move email” y “Create draft”).

Si usas Gmail:

1En el módulo Gmail, campo Connection → pulsa Add.
2Nombre: Gmail-Despacho. Connection type: Gmail. Pulsa Sign in with Google.
3Elige la cuenta Google del despacho → revisa los permisos (leer, mover, crear borradores) → Permitir.
4Vuelves a Make. La conexión queda guardada y seleccionada.

Si usas Outlook (Microsoft 365):

1En el módulo Microsoft 365 Email, campo Connection → Add.
2Nombre: Outlook-Despacho. Pulsa Sign in with Microsoft.
3Acepta los permisos (correo, borradores, carpetas).
4Si es cuenta de empresa puede pedir aprobación del administrador del tenant.
💡 Si el navegador bloquea la ventana emergente, fíjate en la barra de direcciones: hay un icono que permite “Permitir ventanas emergentes para make.com”. Tras permitirlas, vuelve a pulsar Add.
IA Conexión OpenAI · Groq · Gemini (API key)

Cuándo: la primera vez que añades el módulo de IA (Nodo 2 del Módulo 3, “Create a Chat Completion”).

1En el módulo de IA, campo Connection → pulsa Add.
2Nombre: OpenAI-Curso · Groq-Curso · Gemini-Curso (según proveedor).
3Pega tu API Key en el campo correspondiente (no le añadas espacios ni saltos de línea).
4Pulsa Save. La conexión queda disponible en el desplegable para los Nodos 11 y 12.
!
Errores típicos al pegar la key

Invalid API key: sobra un espacio al inicio/fin o la copiaste a medias — vuelve al panel del proveedor y crea otra. insufficient_quota (OpenAI): no tienes saldo, añade en Billing. 403 Forbidden (Gemini): la key se creó en un proyecto sin acceso a la API — créala en un proyecto nuevo.

💡 Si la pierdes o se filtra: entra al panel del proveedor, revoca la antigua y crea una nueva. Luego en Make, edita la conexión y pega la nueva. Todos los escenarios la usarán al instante.

PROMPT 1 · Clasificador
Eres un sistema automático de análisis de correos para una empresa
de administración de fincas.

Debes hacer DOS cosas:
1. Clasificar el email en una categoría
2. Decidir si se debe responder o no

CATEGORÍAS (usa exactamente estos valores):
- contabilidad   → facturas, pagos, recibos, bancos
- incidencias    → averías, problemas técnicos, quejas
- jurídico       → temas legales, contratos, abogados
- documentación  → envío o solicitud de documentos
- otros          → cualquier otro caso

RESPONDER:
- true  → si requiere respuesta (clientes, incidencias, solicitudes)
- false → si NO requiere respuesta (spam, newsletters, automáticos)

⇆ Nodo 4 · Router de 5 categorías

FLOW Flow Control · Router

¿Qué hace? Divide el flujo en 5 caminos según la categoría. Cada camino mueve el email a su carpeta correcta.

Cómo añadirlo: clic en el semicírculo derecho del Nodo 3 → busca Flow Control → elige Router.

Configurar las 5 ramas

El Router crea automáticamente 2 ramas vacías. Añade 3 más con el + de la derecha. Para cada rama, haz clic en la llave inglesa sobre la conexión y configura el filtro:

RamaFilter nameCondición
1Contabilidad{{3.categoria}} · Text contains · contabilidad
2Incidencias{{3.categoria}} · Text contains · incidencias
3Jurídico{{3.categoria}} · Text contains · jurídico
4Documentación{{3.categoria}} · Text contains · documentación
5Otros(deja en blanco — esta es la rama por defecto · marca Fallback route)
💡 La Fallback route de “Otros” captura cualquier email que no encaje en las 4 anteriores. Sin ella, los emails sin categoría clara se pierden.

04Módulo 4 · Variantes

Las 4 versiones del flujo y qué cambia en cada una.

El esqueleto que has construido es el mismo. Lo que cambia son 2 piezas: el correo (Gmail/Outlook) y la IA (OpenAI/Groq/Gemini).

01 Gmail + OpenAI
  • Con confirmación de lectura.
  • IA: GPT (calidad alta, coste por uso).
  • Carpeta destino: Borrador.
  • Para: despachos con volumen y presupuesto.
02 Gmail + Groq
  • Sin confirmación de lectura.
  • IA: Llama 3.1 vía Groq (rápida, gratuita hasta cuota).
  • Carpeta destino: Borradores.
  • Para: empezar sin coste.
03 Outlook + Groq
  • Para Microsoft 365 / Exchange.
  • IA: Groq.
  • Carpeta destino: Borradores (nativa).
  • Para: despachos en Microsoft.
04 Outlook + Gemini
  • Para Microsoft 365 / Exchange.
  • IA: Gemini de Google (plan gratuito generoso).
  • Carpeta destino: Borradores.
  • Para: combinar Outlook con Google AI.
✓
Mi recomendación si empiezas hoy

Si usas Gmail → empieza con el 02 (Gmail + Groq). Si usas Outlook → empieza con el 04 (Outlook + Gemini). Ambos te dan plan gratuito para probar sin gastar un euro.

05Módulo 5 · Activación

Probar, activar y dejarlo trabajando solo.

Ya tienes el escenario montado. Ahora vienen los 3 últimos botones.

1

Botón “Run once”

Abajo a la izquierda. Lanza una pasada de prueba con un email reciente. Mira los círculos verdes encima de cada módulo: indican qué se ejecutó.

2

Comprueba en tu correo

Vete a Gmail/Outlook. ¿Está el email en la carpeta correcta? ¿Tienes un borrador esperando? Si sí, perfecto.

3

Activa la programación

Abajo a la izquierda hay un selector con un reloj. Cámbialo a cada 15 minutos y pulsa el interruptor ON. Listo.

!
Errores comunes en el primer test

Error en el módulo IA: probablemente la API key está mal. Vuelve a generarla. Error en “Move email”: el nombre de la carpeta no coincide. El JSON no se parsea: sube la temperature a 0 y el response_format a JSON object.

06Cierre

Lo has construido tú. Y eso lo cambia todo.

Ahora sabes cómo está hecho. Eso significa que puedes modificarlo, ampliarlo y arreglarlo cuando algo falle. ¿Qué viene después?

  • Ajusta el Prompt 3 con frases típicas de tu despacho.
  • Añade nuevas categorías (proveedores, juntas, morosos…) duplicando ramas del Router.
  • Conecta el flujo con tu CRM o software de fincas para que las incidencias generen tickets.
★
Recordatorio final

La IA es una asistente, no una sustituta. Revisa siempre los borradores antes de enviarlos, especialmente los jurídicos. La automatización te libera tiempo para pensar mejor, no para pensar menos.
